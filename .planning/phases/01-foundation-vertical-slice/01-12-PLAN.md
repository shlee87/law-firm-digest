```yaml
# .github/workflows/daily.yml
name: Daily Digest

on:
  schedule:
    - cron: '0 9 * * *'        # 09:00 UTC = 18:00 KST (OPS-01)
  workflow_dispatch: {}         # manual trigger for smoke tests / gap closure

concurrency:                    # OPS-02 — serialize overlapping runs (Pitfall #5)
  group: digest-pipeline
  cancel-in-progress: false

permissions:
  contents: write               # required by stefanzweifel/git-auto-commit-action

jobs:
  digest:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: pnpm/action-setup@v4
        with:
          version: 9
      - uses: actions/setup-node@v6
        with:
          node-version: lts/*
          cache: 'pnpm'
      - run: pnpm install --frozen-lockfile
      - run: pnpm tsx src/main.ts
        env:
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
          GMAIL_APP_PASSWORD: ${{ secrets.GMAIL_APP_PASSWORD }}
          # RECIPIENT_EMAIL: ${{ secrets.RECIPIENT_EMAIL }}          # optional override (D-05)
          # GMAIL_FROM_ADDRESS: ${{ secrets.GMAIL_FROM_ADDRESS }}    # optional
      - uses: stefanzweifel/git-auto-commit-action@v7
        with:
          commit_message: 'chore(state): update seen items [skip ci]'
          file_pattern: 'state/seen.json'
```

Decisions baked into this YAML:

1. **actions/checkout@v6** — latest as of 2026-04-17 (RESEARCH.md §GitHub Actions versions L168). Default submodules=false; `.gsd-patches` stays untouched (CONTEXT.md code_context note).
2. **pnpm/action-setup@v4** — `[CONFIRM AT PLAN]` flagged in RESEARCH.md A6 and PATTERNS.md L1231. Planner confirms: @v4 is current. If Context7/GitHub releases shows a newer major, pin to that instead.
3. **actions/setup-node@v6** — `node-version: lts/*` resolves to Node 22 (LTS through April 2027 per CLAUDE.md). `cache: 'pnpm'` enables pnpm store caching across runs. **W3**: this combo requires `package.json` to declare `packageManager` (plan 01 Task 1 pins `pnpm@9.15.0`). Pre-flight grep below enforces.
4. **Cron '0 9 * * *'**: OPS-01 (18:00 KST). NOT midnight UTC (congestion avoidance, Pitfall #7).
5. **workflow_dispatch: {}** present to enable manual UI-triggered runs (smoke testing, gap closure, operator debugging).
6. **concurrency.cancel-in-progress: false** — MUST be false. `true` would cancel a running pipeline when a new one starts; `false` queues overlaps — letting the in-flight run complete + commit state before the next starts. Pitfall #5 fix.
7. **permissions.contents: write** — required by git-auto-commit-action@v7 to push state/seen.json back.
8. **Secrets injection via `env:`** — not `inputs:`, not hardcoded. GEMINI_API_KEY and GMAIL_APP_PASSWORD are the two required. RECIPIENT_EMAIL + GMAIL_FROM_ADDRESS commented (optional per D-05).
9. **git-auto-commit-action@v7** — latest as of 2026-04-17. `file_pattern: 'state/seen.json'` ensures only state (not accidentally committed build artifacts) gets pushed. `[skip ci]` in commit message is MANDATORY — without it, the auto-commit triggers a new workflow run, which commits state, which triggers another run — infinite loop.

`.gsd-patches` submodule: default checkout behavior is `submodules: false`. Do NOT add `with: submodules: recursive` — patches are local-dev tooling and should not run in CI.

`DRY_RUN` is NOT injected via `env:` in this workflow. Scheduled production runs must be full-fat. If an operator wants a dry-run in CI, they can use `workflow_dispatch` with a temporary env override, but that's operational, not architectural.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GHA Secrets → workflow env | Secrets cross the GitHub org/repo security boundary into the runner |
| git-auto-commit-action → repo main branch | Action has write access; could in principle push any file (it's bounded by `file_pattern`) |
| workflow trigger → schedule | Inactivity risk: GHA disables scheduled workflows after 60 days with no activity |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Information Disclosure | Secrets leaked via log | mitigate | GHA auto-redacts known secret values in logs; `scrubSecrets` in src/ as belt-and-suspenders. `env:` injection is the canonical path (not inline). Private repo (COMP-04) further reduces radius. |
| T-12-02 | Tampering | Action push loop via missing [skip ci] | mitigate | `commit_message` contains literal `[skip ci]`. Workflow triggers on schedule + workflow_dispatch only — push events do NOT trigger. Belt-and-suspenders. |
| T-12-03 | Denial of Service | GHA 60-day schedule inactivity disable | accept | Each successful run performs a state commit (activity), so the 60-day counter resets daily. If runs fail for 60 consecutive days, workflow disables — operator intervention needed (covered by README operational notes). |
| T-12-04 | Tampering | State race between scheduled + dispatched runs | mitigate | `concurrency: { group: digest-pipeline, cancel-in-progress: false }` serializes. |
| T-12-05 | Repudiation | Silent workflow failure (Pitfall 6) | mitigate | main.ts (plan 11) exits 1 on any FATAL; GHA shows red X. Email delivery error path (plan 09) explicitly throws. Auto-commit action `skip_empty_changes: false` (default true) means even a no-op state write records activity. |
| T-12-06 | Information Disclosure | Secrets file committed | mitigate | plan 01 `.env` in `.gitignore`; `.env.example` is the only committed template. `config/recipient.yaml` contains no secrets (email is semi-public). GHA Secrets never reach filesystem. |
| T-12-07 | Tampering (SSRF) | User controls firm URL → outbound fetch | accept | Only developer-authored `config/firms.yaml` contents are fetched. Phase 1 single firm. Plan 03 zod + plan 05 URL validation already in place. Multi-tenant / user-uploaded YAML is out of scope for Phase 1. |
| T-12-08 | Spoofing | git-auto-commit-action uses compromised credentials | accept | Action uses the default `GITHUB_TOKEN` (ephemeral, scoped to repo). `permissions: contents: write` is the minimum; no `id-token`, no `packages: write`, no `actions: write`. |
| T-12-09 | Denial of Service (cache flakiness) | setup-node@v6 cache: 'pnpm' without packageManager pin | mitigate | W3 pre-flight in Task 1: `grep -c '"packageManager"' package.json` must return 1 before the YAML is written. Without this, GHA cache resolution can produce spurious mismatches on daily runs. |
</threat_model>
