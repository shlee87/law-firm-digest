```typescript
import type { FirmResult, SeenState, NewItem } from '../types.js';
// URLs are ALREADY canonicalized by the scraper (src/scrapers/rss.ts).

export function dedupAll(results: FirmResult[], seen: SeenState): FirmResult[] {
  return results.map((r) => {
    if (r.error) return r;                            // pass-through fetch failures
    const priorFirm = seen.firms[r.firm.id];
    if (!priorFirm) {
      // D-09 first-run bootstrap: seed, don't emit.
      // Note: r.raw is preserved as-is so plan 10 writer can use it to seed state (B1).
      return { ...r, new: [] };
    }
    const seenSet = new Set(priorFirm.urls);
    const fresh: NewItem[] = r.raw
      .filter((item) => !seenSet.has(item.url))
      .map((item) => ({ ...item, isNew: true as const }));
    return { ...r, new: fresh };
  });
}
```

Logical invariants (cite tests against each):
1. **Bootstrap** — seen.firms[id] undefined → new: []. r.raw is unchanged (writer consumes it — B1).
2. **Normal** — seen has some URLs; raw has a mix of seen + unseen → new = raw minus seen
3. **All-seen** — raw items are all in seen → new: []
4. **Empty-raw** — raw: [] → new: []
5. **Error pass-through** — FirmResult with .error → unchanged (no dedup attempt)
6. **Canonical URL assumption** — dedup does NOT re-canonicalize; it compares exact strings (scraper plan 05 is responsible for canonicalizing before dedup sees anything)

DEDUP-03 note ("send only when there's something new"): this plan handles the dedup; main.ts (plan 11) handles the "only send if totalNew > 0" check. The dedup output simply reports new items; the caller is responsible for the skip-email decision.
