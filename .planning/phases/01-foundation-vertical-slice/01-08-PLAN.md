```typescript
import type { FirmResult } from '../types.js';

export function renderHtml(firms: FirmResult[], dateKst: string): string {
  const sections = firms
    .map((r) => {
      const items = r.summarized
        .map(
          (it) => `
      <div style="margin:0 0 16px 0;">
        <div><a href="${escapeAttr(it.url)}">${escapeHtml(it.title)}</a></div>
        ${
          it.summary_ko
            ? `<p style="margin:4px 0 0 0;color:#333;">${escapeHtml(it.summary_ko)}</p>`
            : `<p style="margin:4px 0 0 0;color:#999;font-style:italic;">요약 없음 — 본문 부족</p>`
        }
      </div>`,
        )
        .join('');
      return `<section><h2 style="font-size:18px;margin:24px 0 8px 0;">${escapeHtml(r.firm.name)}</h2>${items}</section>`;
    })
    .join('');

  return `<!doctype html><html><body style="font-family:sans-serif;max-width:680px;margin:0 auto;padding:16px;">
    <h1 style="font-size:22px;">법률 다이제스트 ${escapeHtml(dateKst)}</h1>
    ${sections}
    <footer style="margin-top:32px;color:#888;font-size:12px;">AI 요약 — 원문 확인 필수</footer>
  </body></html>`;
}

function escapeHtml(s: string): string {
  return s.replace(/[&<>]/g, (c) => ({ '&': '&amp;', '<': '&lt;', '>': '&gt;' })[c]!);
}

function escapeAttr(s: string): string {
  return s.replace(/["&<>]/g, (c) => ({ '"': '&quot;', '&': '&amp;', '<': '&lt;', '>': '&gt;' })[c]!);
}
```

src/compose/digest.ts:

```typescript
import { formatInTimeZone } from 'date-fns-tz';
import { renderHtml } from './templates.js';
import type { FirmResult, EmailPayload } from '../types.js';

export function composeDigest(
  results: FirmResult[],
  recipient: string,
  fromAddr: string,
  now: Date = new Date(),
): EmailPayload {
  const firmsWithNew = results.filter((r) => r.summarized.length > 0);
  const dateKst = formatInTimeZone(now, 'Asia/Seoul', 'yyyy-MM-dd');
  const itemCount = firmsWithNew.reduce((n, r) => n + r.summarized.length, 0);
  const subject = `[법률 다이제스트] ${dateKst} (${firmsWithNew.length} firms, ${itemCount} items)`;
  const html = renderHtml(firmsWithNew, dateKst);
  return { subject, html, to: recipient, from: fromAddr };
}
```

Design notes:
- `composeDigest` accepts optional `now: Date = new Date()` so tests can inject a deterministic date.
- D-06 subject format: exact literal, Korean bracket string `[법률 다이제스트]`, ISO date, `(N firms, M items)`. Accepts the "1 firms" grammar issue per CONTEXT.md D-06.
- D-07 HTML: minimal inline CSS; h1/h2 sizing; item block is title-link + Korean summary paragraph. No external images. No dark mode. No firm icons.
- **B3 null-summary placeholder**: when `it.summary_ko` is null (source: Gemini failure OR main.ts B3 'skipped' bypass), render the literal Korean string `요약 없음 — 본문 부족` in italic grey. Covers both origin paths with a single user-facing message because the underlying cause in both cases is "no body content to summarize from".
- D-08 plaintext alternative: NOT added. `EmailPayload` has no `text` field (plan 03 types.ts confirms).
- Firms with zero summarized items are excluded from the body and subject count (they would clutter the email).
- `formatInTimeZone(now, 'Asia/Seoul', 'yyyy-MM-dd')`: KST date for the subject. Plan 11 main.ts passes `new Date()` implicitly; tests override.
- Escape functions are hand-rolled (3–4 chars each). For richer escape, add `he` npm dep — but Phase 1 budget says no (RESEARCH.md PATTERNS.md L683).

Test approach: snapshot test the rendered HTML against a fixed input. The test's `expect(html).toMatchInlineSnapshot(...)` captures the current output; future regressions fail loud.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scraped title string → HTML body | Scraped content (title) crosses into HTML context — XSS risk |
| Gemini summary text → HTML body | LLM output crosses into HTML — XSS if model emits raw tags |
| scraped URL → href attribute | URL crosses into HTML attribute — injection risk |
| date string → h1 text | Developer-controlled date, minimal risk but escape anyway |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering (XSS) | RawItem.title → HTML | mitigate | `escapeHtml` applied at every insertion (unit-tested with `<Article>` → `&lt;Article&gt;`). Renderer never concatenates raw title into HTML. |
| T-08-02 | Tampering (XSS) | summary_ko → HTML | mitigate | `escapeHtml` applied. Gemini outputs JSON and zod validates shape (plan 06), so practical exploit requires the model to both emit valid structured JSON AND include raw HTML in summary_ko — still neutralized by escape. |
| T-08-03 | Tampering (attribute injection) | url → href | mitigate | `escapeAttr` with `"` → `&quot;` prevents attribute-close attack. URL is already canonicalized (plan 04) to valid URL form, so the `javascript:` scheme is less of a concern — however if an attacker crafted a `javascript:` URL upstream, escapeAttr does not filter schemes. Defense-in-depth: plan 05 rss.ts consumes RSS with standard `<link>` element which practically always produces http(s)://. |
| T-08-04 | Information Disclosure | email renders to user | accept | Recipient is the single user; no multi-tenant risk. |
| T-08-05 | Tampering (XSS) via Subject | mail client rendering | accept | Subject line doesn't contain user-controlled content in Phase 1 (only developer-authored Korean literal + ISO date + integers). No escape needed. |
| T-08-06 | Repudiation (null-summary ambiguity) | user sees blank where summary should be | mitigate | B3 placeholder `요약 없음 — 본문 부족` is rendered in italic grey for ALL null-summary cases. Test asserts both 'failed' and 'skipped' origin items produce the same visible string. |
</threat_model>
