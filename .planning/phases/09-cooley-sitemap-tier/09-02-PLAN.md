<decisions>
**Title derivation — Option A (URL slug, title-cased):** Sitemap XML does not carry article titles. `scrapeSitemap` derives a fallback title from the URL pathname's last non-empty segment (slug), with hyphens → spaces, and Title-Case applied. Example: `https://www.cooleygo.com/share-incentives-employees-private-uk-companies/` → `'Share Incentives Employees Private Uk Companies'`.

Rationale (research Open Question 1 resolution):
- Option A is cheapest (no extra network I/O) and stays within D-02's "no sitemap-specific detail-scraping logic" constraint.
- Gemini summaries use the actual article body (via enrichBody's Playwright branch), so title quality is cosmetic — it only shows as the digest section header.
- Option B (page.title() during detail fetch) and C (empty-title tolerance) are explicitly rejected: B violates D-02, C is unsupported by downstream compose/digest.ts.
- Known downside: ugly Title-Case on acronyms (`UK → Uk`). Accepted for Phase 9; future title-quality improvement is a separate backlog item.

**Browser-HTTP fetch (D-16 revision):** Uses `context.request.get(firm.url, { timeout })`. This replaces the original D-16 plan of `decodeCharsetAwareFetch`, which the 2026-04-20 live probe showed to get HTTP 403 on `post-sitemap.xml` from undici but HTTP 200 from Playwright's browser-HTTP stack (3/3 probes, same USER_AGENT). Plumbing through Playwright is required — no exception.

**cheerio xmlMode syntax:** Use `cheerio.load(xml, { xml: true })` (shorthand). Equivalent to `{ xmlMode: true }` in cheerio 1.2.0. Selector syntax inside sitemap documents is case-sensitive — use `$('url')` not `$('URL')`, and `<loc>` / `<lastmod>` lowercase.

**Unparseable-date drop:** `new Date(lastmodRaw).getTime()` returns `NaN` for garbage. We drop those alongside missing-lastmod entries (same effect on downstream sort). No warning logs required for Phase 9 — quiet drop is fine; volume observability is Phase 10's job.

**DEFAULT_LATEST_N constant:** Lives at module top of `src/scrapers/sitemap.ts`. Value: 10. Keeps the default with the consumer so schema stays explicit (see plan 01 Decisions).

    // Sitemap scraper — Phase 9 tier-4 path (D-01 / D-16 revision).
    //
    // Sibling to src/scrapers/rss.ts, html.ts, jsRender.ts. Consumes a FirmConfig
    // where type='sitemap' + url points at a WordPress post-sitemap.xml. Emits
    // the same RawItem[] contract so downstream dedup / enrichBody / summarize /
    // compose stay tier-agnostic.
    //
    // D-16 REVISION (2026-04-20 live probe): Cloudflare extends managed-challenge
    // to the sitemap XML endpoint when hit via undici (native fetch). Playwright's
    // browser-HTTP stack (context.request.get) reliably returns HTTP 200 with the
    // same LegalNewsletterBot UA. This module MUST use Playwright for the XML
    // fetch; decodeCharsetAwareFetch is NOT viable for Cooley.
    //
    // Browser lifecycle (D-05 mirror):
    //   - browser is an INJECTED parameter, NOT a module singleton. run.ts owns
    //     launch()+close(); this module owns only the per-firm BrowserContext.
    //   - context.newContext({ userAgent }) per firm — cookies/storage isolated.
    //   - context.close() in finally {} — zombie-context prevention.
    //
    // Error message shapes (COUPLED to classifyError regex in
    // src/compose/templates.ts; DO NOT modify without updating classifier in lockstep):
    //   - 'scrapeSitemap {firm.id}: HTTP {status}' — XML fetch non-OK.
    //   - 'scrapeSitemap {firm.id}: malformed XML (no <urlset> root)' — cheerio
    //     loaded but root tag absent; likely HTML error page returned with 200.
    //   - 'scrapeSitemap {firm.id}: zero items extracted' — XML parsed OK but
    //     every <url> entry failed the loc+lastmod filter (D-10 drop policy).
    //
    // Title derivation (Phase 9 Option A, CONTEXT Decision resolved by plan 02):
    //   - Sitemap XML has no <title>. We derive a slug title from the URL pathname
    //     last segment, hyphens → spaces, Title-Case applied. Gemini summaries
    //     use real body text via enrichBody's Playwright branch, so title is
    //     cosmetic. Future enhancement: detail-page title via enrichBody is
    //     intentionally out of scope (D-02: no sitemap-specific detail logic).

    import type { Browser } from 'playwright';
    import * as cheerio from 'cheerio';
    import { USER_AGENT } from '../util/logging.js';
    import { canonicalizeUrl } from './util.js';
    import type { FirmConfig, RawItem } from '../types.js';

    const FETCH_TIMEOUT_MS = 20_000;
    export const DEFAULT_LATEST_N = 10;

    /**
     * Fetch a sitemap XML via Playwright browser-HTTP, parse it with cheerio
     * xml mode, and return the top-N most-recent entries as RawItem[].
     *
     * @param firm FirmConfig with type='sitemap' and url pointing at sitemap XML.
     *             firm.latest_n caps the returned RawItem count (default 10).
     * @param browser A launched chromium Browser instance. Owned by run.ts.
     * @throws Error with classifier-coupled shape on fetch / parse / zero-items.
     */
    export async function scrapeSitemap(
      firm: FirmConfig,
      browser: Browser,
    ): Promise<RawItem[]> {
      const context = await browser.newContext({ userAgent: USER_AGENT });
      let xml: string;
      try {
        const res = await context.request.get(firm.url, {
          timeout: FETCH_TIMEOUT_MS,
        });
        const status = res.status();
        if (!res.ok()) {
          throw new Error(`scrapeSitemap ${firm.id}: HTTP ${status}`);
        }
        xml = await res.text();
      } finally {
        await context.close();
      }

      const $ = cheerio.load(xml, { xml: true });
      if ($('urlset').length === 0) {
        throw new Error(
          `scrapeSitemap ${firm.id}: malformed XML (no <urlset> root)`,
        );
      }

      type Entry = { loc: string; lastmodMs: number };
      const entries: Entry[] = [];
      $('url').each((_: number, el) => {
        const loc = $(el).find('loc').first().text().trim();
        const lastmodRaw = $(el).find('lastmod').first().text().trim();
        if (!loc) return;
        if (!lastmodRaw) return; // D-10: drop missing lastmod
        const t = new Date(lastmodRaw).getTime();
        if (Number.isNaN(t)) return; // drop unparseable dates
        entries.push({ loc, lastmodMs: t });
      });

      if (entries.length === 0) {
        throw new Error(`scrapeSitemap ${firm.id}: zero items extracted`);
      }

      entries.sort((a, b) => b.lastmodMs - a.lastmodMs);
      const topN = entries.slice(0, firm.latest_n ?? DEFAULT_LATEST_N);

      return topN.map(({ loc, lastmodMs }) => ({
        firmId: firm.id,
        title: titleFromUrl(loc),
        url: canonicalizeUrl(loc, firm.url),
        publishedAt: new Date(lastmodMs).toISOString(),
        language: firm.language,
      }));
    }

    /**
     * Derive a fallback title from a URL's pathname last segment.
     * Example: 'https://www.cooleygo.com/share-incentives-employees-uk/' →
     *          'Share Incentives Employees Uk'
     * Falls back to the full URL if the slug can't be derived.
     */
    function titleFromUrl(url: string): string {
      try {
        const u = new URL(url);
        const parts = u.pathname.split('/').filter((p) => p.length > 0);
        const slug = parts[parts.length - 1] ?? '';
        if (!slug) return url;
        return slug
          .split('-')
          .filter((w) => w.length > 0)
          .map((w) => w[0].toUpperCase() + w.slice(1).toLowerCase())
          .join(' ');
      } catch {
        return url;
      }
    }

## Security Threat Model (ASVS L1)

**Scope:** New scraper module reads remote XML through Playwright's browser-HTTP stack. Threats center on (a) untrusted XML input, (b) URL handling, (c) DOS vectors from pathological payloads.

| Threat | Severity | Mitigation |
|--------|---------|-----------|
| T1: XXE (XML External Entity) injection via `<!DOCTYPE>` declaration pulling internal files/HTTP | MEDIUM | cheerio wraps htmlparser2 which does NOT resolve external entities by default. `xml:true` mode activates XML parsing rules but still no entity resolution. Manually verify by testing with `<!DOCTYPE x [ <!ENTITY test SYSTEM "file:///etc/passwd"> ]>` in a follow-up spike if needed — not blocking for Phase 9. |
| T2: DOS via huge XML payload (100MB sitemap) exhausting memory | LOW | Playwright `context.request.get` respects the 20s timeout via `FETCH_TIMEOUT_MS` — huge-payload downloads time out before completing. Cooley's real sitemap is ~41KB; 100× inflation from a hostile CDN is implausible. If observed, future phase adds response-size cap. |
| T3: Zip-bomb-style compressed XML (gzip content-encoding expanding to 10GB) | LOW | Playwright handles decompression transparently, but in-memory to string. Same 20s timeout bounds exposure. Not a Phase 9 blocker. |
| T4: Malicious `<loc>` URL smuggling — javascript:/data:/file: schemes in sitemap | MEDIUM | canonicalizeUrl uses `new URL(...)` which throws on non-http(s) schemes, and the downstream enrichBody Playwright fetch rejects non-http(s) in page.goto. Invalid URLs propagate as per-item failures (try/catch in enrichBody per-item) — no credential leak path. |
| T5: XSS via malicious title derivation (slug contains HTML entities) | LOW | titleFromUrl uses `new URL().pathname` which is already URL-decoded but not HTML-unescaped. Downstream compose/digest.ts uses `escapeHtml` before rendering titles into email HTML. Invariant preserved across tiers. |
| T6: Unbounded sort on a 100K-entry sitemap | LOW | Array.sort is O(n log n). 233 entries (Cooley today) or even 10K (hypothetical large sitemap) is trivial. `firm.latest_n` caps the OUTPUT, not the sort — but 10K entries sort in <5ms; acceptable. |
| T7: Timing side-channel via fetch duration revealing CF challenge state | LOW | Scraper runs once per day; attacker cannot observe. Not a realistic threat. |
| T8: Browser context escape / zombie context accumulation | LOW | `finally { await context.close(); }` is unconditional. Unit test 7/8 asserts context.close is called even when fetch throws. |

**No HIGH severity threats.** XXE (T1) is the closest and is mitigated by parser default.

**Residual risk:** XXE mitigation is parser-implementation-dependent. A cheerio major-version upgrade could theoretically change defaults. Mitigation: CI depends on cheerio 1.2.0 pin; upgrades are opt-in via package.json.
</threat_model>
