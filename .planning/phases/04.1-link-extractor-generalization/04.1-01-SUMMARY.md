## The Three Modes (YAML shapes)

### Mode 2 (legacy — unchanged): string `link`

```yaml
selectors:
  list_item: .post-prime
  title: a.text
  link: a.text        # plain CSS selector; extractor reads href as-is
  date: .foot-item.posted
```

Used today by: cooley (rss has no selectors), shin-kim, yulchon, logos, skadden, lee-ko (disabled placeholder), yoon-yang (disabled placeholder), latham (disabled placeholder).

### Mode 3 (DEPRECATED — backward-compat): `link_onclick_regex` + `link_template`

```yaml
selectors:
  list_item: ul.notice_list > li
  title: .notice_txt .title
  link_onclick_regex: "goDetail\\('(\\d+)','(\\d+)'\\)"
  link_template: "https://www.kimchang.com/ko/insights/detail.kc?sch_section={1}&idx={2}"
  date: .notice_date
```

Used today by: kim-chang, bkl. Kept valid indefinitely. New firms should use Mode 1 instead.

### Mode 1 (NEW — Phase 4.1): object `link`

```yaml
selectors:
  list_item: ul#contentsList > li
  title: .tit
  link:
    selector: a
    attribute: href        # optional, defaults to 'href'
    regex: "doView\\((\\d+)\\)"
    template: "/kor/insights/newsletter/{1}"
  date: .article_date
```

Covers: any attribute (href, onclick, data-id, custom), optional regex extraction, optional template substitution with capture groups `{1}`, `{2}`, etc. Unused regex/template → plain attribute read (equivalent to Mode 2).

## yoon-yang — paste-ready config for plan 04-08

**Blocker removed.** Previously `yoon-yang: enabled: false` because `parseListItemsFromHtml` couldn't resolve `href="javascript:doView(N)"` without an `onclick` attribute. Now expressible:

```yaml
- id: yoon-yang
  name: 법무법인 화우
  language: ko
  type: js-render
  url: https://www.yoonyang.com/kor/insights/newsletters
  timezone: Asia/Seoul
  enabled: true          # ← now safe to enable after this plan lands
  wait_for: "ul#contentsList > li"
  selectors:
    list_item: "ul#contentsList > li"
    title: ".tit"         # NOT .title — see 04-07-PROBE-RESULTS.md deviation table
    link:
      selector: "a"
      # attribute defaults to 'href'
      regex: "doView\\((\\d+)\\)"
      template: "/kor/insights/newsletter/{1}"
```

Test vector: yoon-yang's `<a href="javascript:doView(14735)">` with above config → canonical URL `https://yoonyang.com/kor/insights/newsletter/14735` (www. stripped by `canonicalizeUrl` per DEDUP-02).

Verified by Task 3 test "Mode 1 (object link, attribute=href + regex + template): yoon-yang pattern" — the canonical URL resolution is locked.

## When to migrate kim-chang and bkl

**Recommendation: triggered, not forced.** Their current `link_onclick_regex` + `link_template` config works and is covered by Mode 3. Migrate when:

1. Either firm changes their site such that a selector needs adjustment (cheaper to rewrite then than touch-for-touch).
2. A future plan formally sunsets Mode 3 (not on the near-term roadmap).
3. A future contributor prefers the Mode 1 shape for consistency when adding a similar KR onclick-style firm.

Migration shape:

```yaml
# Before (Mode 3)
link_onclick_regex: "goView\\('(\\d+)'\\)"
link_template: "/law/insight/informationView.do?infoNo={1}&lang=ko"

# After (Mode 1)
link:
  selector: "a[onclick]"
  attribute: "onclick"
  regex: "goView\\('(\\d+)'\\)"
  template: "/law/insight/informationView.do?infoNo={1}&lang=ko"
```

Equivalent runtime behavior — both traverse `a[onclick]`, apply regex to the onclick string, substitute capture group into template.

## Decisions Made

- **Wide union over discriminated union:** No `type: 'string' | 'object'` tag on selectors.link. Zod's runtime `typeof` + TS narrowing handle the two shapes without YAML clutter. Adding a tag would force every existing firm's YAML to add a redundant `link.type: 'css'` line.
- **Legacy fields kept valid, not removed:** Migration is triggered (firm site change, future sunset plan), not forced now. Zero churn for kim-chang and bkl.
- **`attribute: 'href'` default:** Zod `.default('href')` means the common case (plain href read via object form for symmetry with Mode 1's regex/template support) requires no redundant `attribute: href` line.
- **Transient typeof narrow in Task 1 (util.ts):** To keep `pnpm typecheck` green at the Task 1 commit boundary (before Task 2 landed the real refactor), a minimal `typeof selectors.link === 'string'` narrow was added to the pre-existing string branch. Task 2 then replaced the entire branch block with `extractLinkUrl(...)` — so the narrow was transient scaffolding, not a permanent code path. Both commits pass typecheck + tests independently (build green at every boundary).
