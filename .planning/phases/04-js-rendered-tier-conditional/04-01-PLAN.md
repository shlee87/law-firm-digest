4. **Add a `.superRefine()` to FirmSchema** AFTER `.strict()` to cross-validate `type` and `wait_for`:

```typescript
.superRefine((firm, ctx) => {
  if (firm.type === 'js-render') {
    if (!firm.wait_for || firm.wait_for.length === 0) {
      ctx.addIssue({
        code: z.ZodIssueCode.custom,
        message: 'firms[].wait_for is required when type === "js-render"',
        path: ['wait_for'],
      });
    }
  } else {
    if (firm.wait_for !== undefined) {
      ctx.addIssue({
        code: z.ZodIssueCode.custom,
        message: 'firms[].wait_for is only valid when type === "js-render"',
        path: ['wait_for'],
      });
    }
  }
})
```

**IMPORTANT:** zod 4 requires `superRefine` to be called on the object BEFORE `.strict()` converts the schema type, OR it must be called on the post-`.strict()` result. With zod v4 (`^4.3.6` in package.json), `.strict().superRefine()` is valid — superRefine returns a ZodEffects which preserves strict-mode semantics. Verify by running the Test 9 regression in task 3 (unknown field still rejected).

5. **Do NOT modify** `FirmsConfigSchema` (line 57-61) or `RecipientSchema` (line 72-76) — they are unchanged.

6. **Preserve** the existing `.refine()` on the `selectors` subobject (lines 42-49) verbatim — it's orthogonal to the new superRefine.

describe('FirmSchema (Phase 4 js-render extensions)', () => {
  const jsRenderBase = {
    id: 'test-js-firm',

    language: 'ko' as const,
    type: 'js-render' as const,

    const r = FirmSchema.safeParse({
      ...jsRenderBase,
      wait_for: 'ul#contentsList > li',
      selectors: { list_item: 'ul#contentsList > li', title: '.title', link: 'a' },
    });
    expect(r.success).toBe(true);
  });

  it('rejects js-render firm with no wait_for field', () => {
    const r = FirmSchema.safeParse({
      ...jsRenderBase,
      selectors: { list_item: 'li', title: '.t', link: 'a' },

    expect(JSON.stringify(r.error?.issues)).toContain('wait_for is required when type');

    const r = FirmSchema.safeParse({

    });
    expect(r.success).toBe(false);
  });

  it('rejects rss firm that mistakenly includes wait_for', () => {
    const r = FirmSchema.safeParse({
      id: 'bad-rss',
