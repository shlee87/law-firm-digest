# v1.0 data quality audit — post-UAT critical findings

## 요약

v1.0은 **code-path level로는 PASS**(180+ unit tests, 모든 Success Criteria code 검증)이지만, **실제 데이터 품질 관점에서 심각한 결함 다수 발견**. UAT demo 과정에서 nks4860@gmail.com으로 bkl 9개 아이템이 담긴 실제 digest를 받아 확인한 결과, 모든 아이템의 한국어 요약이 **동일한 "BKL 로펌 전반 소개" 템플릿으로 hallucinate** 되어 있었음.

근본 원인 전수 조사(2026-04-19) 결과, html-tier 6개 firm 중 실질적으로 정상 동작하는 것이 **하나도 없음**.

## 근본 원인 공통 패턴

대부분의 Korean 로펌 사이트가 **SPA + JS-hydrated detail pages**를 사용. Phase 2 설계는 이를 고려하지 않음:
- html tier는 list page의 static HTML만 파싱 (OK)
- enrichBody는 detail URL을 **static fetch로만** 가져옴 (static HTML에 실제 content 부재 → SPA routing 필요)
- js-render tier의 detail fallback은 `r.firm.type === 'js-render'`일 때만 트리거 (bkl 같은 html tier는 해당 안 됨)
- Gemini가 생성 단계에서 방어막 없이 hallucinate

## 권장 해결 경로 — v1.1 Phase cycle

### Phase 1 (v1.1-01): 진짜 동작 probe + selector audit
- 각 firm의 list page fetch/selector probe → 실제 item 뽑히는지 확인 (shin-kim, logos, skadden)
- 각 firm의 detail URL Playwright probe → SPA 여부 판정 (bkl, kim-chang 포함)
- js-render 4 firms도 detail body 실제로 뽑히는지 확인
- 출력: `.planning/phases/v1.1-01-firm-audit/01-AUDIT.md` — firm-by-firm 진단표

### Phase 2 (v1.1-02): SPA-detail 대응 tier 확장
- 현재 enrichBody의 Playwright fallback 조건을 `type === 'js-render'`에서 firm-config flag `detail_tier: 'js-render' | 'static'`로 확장
- html tier firm도 `detail_tier: 'js-render'` 설정 가능하게 schema 변경
- bkl/kim-chang 이 flag 켜기

### Phase 3 (v1.1-03): Gemini hallucination guard
- `src/summarize/gemini.ts` prompt에 명시적 guard 추가:
  - "If body content appears to be generic firm overview/navigation (not article-specific), return summary_ko = title verbatim and confidence = 'low'"
  - "If body is empty or shorter than 100 chars, same rule"
- Fixture tests: generic body → title-verbatim, real body → real summary
- Optional: post-summarize dedup — 같은 firm 내에서 첫 50 chars가 동일한 summary가 3개 이상이면 warning/flag

### Phase 4 (v1.1-04): Cooley sitemap tier (이미 별도 backlog에 있음 — cooley-cf-bypass.md)

### Phase 5 (v1.1-05): Observability 강화
- Step summary에 body 품질 지표 추가:
  - 평균 body length per firm
  - "generic body detected" 개수 (hallucination guard 트리거 수)
  - Gemini confidence 분포 (high/medium/low 비율)
- 이상 firm 자동 감지 → footer or step summary에 경고
